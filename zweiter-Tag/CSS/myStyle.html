Einführung CSS

Wir bringe ich CSS in mein Dokument?

  • Inline-Style - hält sich nicht an alle CSS-Syntax Regeln sondern mischt sie mit HTML
  • den style-Tag
  • Eine eigene *.css Datei

Inline-Style

Wir sprechen dann vom inline-Styling wenn wir das style-Attribut im zu stylenden Tag benutzen

Kontrapunkte Inline-Styling

  • Mischen von Struktur und Styling
  • Nicht wiederverwendbar, da wir es für jedes Element wieder schreiben müssen

Positiver Aspekt von Inline-Styling

Auch hier wird das Styling mit dem HTML-Dokument geladen, aber es macht die Datei größer und somit auch langsamer

Style Tag

Der style-Tag wird im head-Tag gesetzt und funktioniert wie ein normales Stylesheet

Kontrapunkte

  • Mischen von CSS und HTML
  • Wiederverwendbarkeit ist zwar innerhalb des Dokumentes gegeben, aber müsste zum teilen mit anderen Elementen immer wieder kopiert werden

Positiver Aspekt von Style-Tag

Auch hier wird das Styling mit dem HTML-Dokument geladen, aber es macht die Datei größer und somit auch langsamer

Eigene CSS-Datei

Gilt als Best Practice

um so vorzugehen brauchen wir einmal eine css Datei und einen link-Tag der auf diese verweist

Positiver Aspekt

  • Wiederverwendbarkeit über mehrere Dokumente und Webseiten hinweg möglich
  • Klare Trennung der Aufgaben

Kontrapunkte

  • Langsamere Ladezeit, da zusätzlich zum HTML Dokument auch eine CSS Datei geladen werden muss
